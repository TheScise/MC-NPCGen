Prompt 1 — Add Manual Editing + Character Persistence
# Feature Request: Manual Editing Mode + Character Persistence

The NPC generator is currently focused on random slot-machine character creation.

I want to add a second workflow:

1. Live random generation (existing)
2. Manual NPC creation/editing (new)

Do not remove or redesign the current slot machine experience.

The random generation flow should remain the default and most entertaining path.

---

## Goals

Allow the user to manually adjust any generated NPC before saving or using it.

The user should be able to:

- Roll a character
- Adjust details manually
- Save the final version

---

# Character Editor

Add an "Edit Character" mode.

This should allow editing:

- Name
- Race
- Nationality
- Class
- Flaw
- Quest

Optional fields:

- Gender
- Location
- Lore
- Notes

The editor should use the same character state as the random generator.

Example workflow:

1. Roll NPC

Harold
Dwarf
Scottish
Potato Farmer
Hates Moon

2. Click Edit

Change:

Name:
Harold → Gregor

Quest:
Defeat Lich → Find Lost Crown

3. Save

---

# Character Data Model

Extend the character model.

Current:

{
name,
race,
class,
flaw,
quest
}

Add:

{
id,
createdAt,
updatedAt,
lore,
location,
notes
}

Use generated IDs.

Do not add a database.

---

# Persistence

Create a simple storage layer.

Use localStorage initially.

Create a reusable utility:

lib/storage.ts

Responsibilities:

- saveCharacter()
- loadCharacter()
- updateCharacter()
- deleteCharacter()
- getCharacters()

Keep storage logic separate from UI.

---

# UI Requirements

Maintain the existing dark RPG aesthetic.

Do not make this look like an admin dashboard.

The edit interface should feel like a character sheet.

Example:

====================

HAROLD

Race:
[Dwarf]

Class:
[Potato Farmer]

Flaw:
[Hates the Moon]

Lore:
[Text area]

[Save Character]

====================

---

# Testing

Verify:

- Existing random roller still works
- Overlay still updates
- Manual changes update overlay
- Refreshing the page preserves saved characters

Keep the implementation simple.

My order recommendation:
